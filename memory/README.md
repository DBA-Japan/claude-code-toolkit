# メモリシステム

Claude Code のメモリシステムは、セッションを跨いで知識を蓄積する仕組みです。

## 仕組み

```
MEMORY.md（インデックス）
  ├── feedback_testing.md     ← 「DBをモックするな」
  ├── project_v2_release.md   ← 「v2は4月リリース」
  ├── user_role.md            ← 「React専門のエンジニア」
  └── reference_linear.md     ← 「バグはLinearで管理」
```

- `MEMORY.md` は常にコンテキストに読み込まれるインデックス
- 個別ファイルは必要な時だけ参照される
- 200行以内に収める（超過分は切り捨て）

## ベストプラクティス

1. **MEMORY.md はリンク集にする** — 本文を書かない。個別ファイルへのポインターだけ
2. **1トピック1ファイル** — 巨大なファイルより小さなファイルの方が管理しやすい
3. **frontmatter を必ず書く** — `name`, `description`, `type` の3つ
4. **`/learn-eval` で品質評価してから保存** — 無駄なメモリを増やさない
5. **定期的に整理** — 古い・不要なメモリは削除する

## メモリに保存すべきもの

- ユーザーの修正（「こうして」「こうしないで」）
- プロジェクトの決定事項（次のセッションが蒸し返さないように）
- 外部リソースの場所（URL、ファイルパス）
- 繰り返し使う設定・ルール

## メモリに保存すべきでないもの

- コードから読み取れる情報（アーキテクチャ、ファイル構成）
- git log で分かる情報（最近の変更、誰が何をしたか）
- 一時的な情報（今のセッションだけで使う）
- CLAUDE.md に書いてある情報（重複）

詳しくは `guides/memory-system.md` を参照。
