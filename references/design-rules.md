# Web デザインルールブック

AI っぽくならない Web デザインの禁止事項・必須事項・チェックリスト。

---

## 絶対禁止

| NG | 理由 |
|----|------|
| 均一カード（同サイズ 3 カード横並び） | AI テンプレの象徴 |
| 丸/ドットのカーソルフォロワー | 安っぽい |
| 虹色の細い線トレイル | 色は鮮やかでいいが虹色 7 色は NG |
| 紫→青グラデーション背景 | AI っぽさの代名詞 |
| グラデーション文字 | 見出しに勝手にグラデ適用 NG |
| ✕/✓ マーク（Before/After） | AI っぽい → 取り消し線+アンダーラインで代替 |
| Inter / Roboto / Arial フォント | AI デフォルト |
| ピンク/紫系グラデーション | AI っぽい |

---

## 必須事項

| 項目 | 詳細 |
|------|------|
| **背景に動き** | ブロブ・テクスチャ・オーロラ等。白でもダークでも動きが欲しい |
| **改行制御** | PC/SP 両方で `<br class="pc-br">` `<br class="sp-br">`。「。」だけ落ちるのは最悪 |
| **モバイルファースト** | 375px で全テキスト改行チェック |
| **テクスチャ** | SVG ノイズ、ドットパターン、グレイン等 |
| **非対称グリッド** | `2fr 1fr` や `0.9fr 1.1fr 0.9fr` で均一を崩す |
| **写真の形バラバラ** | 丸・楕円・正方形・長方形を混ぜる（SmartHR 型） |

---

## テキストハイライト

```css
.highlight {
  background: rgba(255, 85, 0, 0.15);
  padding: 2px 4px;
  border-radius: 2px;
}
```

---

## AI っぽさ回避チェックリスト

- [ ] フォントは Inter/Roboto ではない
- [ ] グラデーション文字を使っていない
- [ ] カードのパディング/角丸が均一ではない
- [ ] 紫→青グラデーション背景がない
- [ ] レイアウトに意図的なズレ・非対称性がある
- [ ] テクスチャ・ノイズ背景を含む
- [ ] 短い尖った文がある（長文ダラダラ NG）
- [ ] ✕/✓ マークを使っていない

---

## カード代替レイアウト（7 パターン）

1. **Bento Grid** — Apple/Figma 風。不均等サイズのカードを組み合わせ
2. **テキスト+余白** — Linear/Vercel 風。カードを使わず余白で区切る
3. **タイムライン** — 縦線+ジグザグ。Before/After やプロセス表示に
4. **展開型アコーディオン** — クリックで展開。情報量が多い時
5. **横スクロール** — GSAP ScrollTrigger pin。ギャラリーに最適
6. **Before/After 分割** — ドラッグスライダーで比較
7. **数字ドン** — 画面いっぱいに 1 つの大きな数字。インパクト重視

## マウスエフェクト

| NG | OK |
|----|----|
| 丸ドット | ぼやけた飛行機雲トレイル（幅 50-100px + blur + フェードアウト） |
| 虹色細線 | ガウシアン減衰の衝撃波（触れた場所だけ波紋） |
| 均一に動く | 近くだけ反応、遠くは無視 |

## デザインパターン自動選定

### Step 1: 業種で大枠

| 業種 | トーン | 動き |
|------|--------|------|
| AI/テック | ダーク 70%+光 30% | 多め（パーティクル、シンボル） |
| SaaS/B2B | ライト or ダーク | 中程度（フェードイン） |
| コンサル/研修 | ダーク+アクセント 1 色 | 控えめ（信頼重視） |
| 旅行/ホテル | ライト、写真主役 | 写真遷移（ズーム、パララックス） |
| クリエイティブ | 自由（個性重視） | 多め（WebGL、カーソル） |

### Step 2: 会社の「人格」で細部

| 大事にしていること | 反映 |
|-------------------|------|
| 革新・先進性 | ダーク、パーティクル、WebGL |
| 信頼・堅実 | 余白、ミニマル、実績数値 |
| 親しみやすさ | 丸い形、暖色、イラスト |
| 高級感 | 黒+金/白、余白極大 |

### Step 3: ターゲット顧客で調整

| ターゲット | 調整 |
|-----------|------|
| 大企業担当者 | 派手すぎない。「怪しくない」ライン |
| スタートアップ | 攻めた OK。WebGL、インタラクション多め |
| 一般消費者 | 分かりやすさ最優先。モバイルファースト |
| 高齢層 | フォント大きめ、動き最小限 |
