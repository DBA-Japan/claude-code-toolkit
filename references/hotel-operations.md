# 宿泊施設運営リファレンス

一棟貸し・バケーションレンタルの運営で Claude Code が役立つ場面と、具体的なワークフロー。

---

## OTA（Online Travel Agency）運用

### Airbnb
- **手数料**: ホスト 3% + ゲスト ~14%（または ホスト 15-18% 一括）
- **スーパーホスト条件**: 4.8★以上、返信率 90%以上、キャンセル率 1%未満
- **説明文の最適化**: `/ota-listing` でタイトル・説明文・キーワードを生成
- **写真**: 最低 20 枚。1 枚目がカバー写真（最重要）

### Booking.com
- **手数料**: 15%（地域による）
- **Genius プログラム**: レビュー高評価で上位表示
- **説明文**: 設備情報を箇条書きで詳細に

### 自社サイト直予約
- **手数料ゼロ** → 最も利益率が高い
- **課題**: 集客。SEO + SNS + リピーター施策が必要
- **予約フォーム**: Formsubmit / Google Forms + GAS でメール通知

## 予約管理の効率化

### CSV エクスポート → 自動集計（AirHost なし）
```
1. Airbnb/Booking.com の管理画面から CSV ダウンロード
2. Claude Code にドラッグ&ドロップ（またはパス指定）
3. 「この予約データを分析して月次レポートを作って」
```

### AirHost 導入済みの場合
- API 経由で全 OTA のデータを一元管理
- iCal 同期でダブルブッキング防止
- Claude Code から API 連携でデータ取得も可能

## 料金設定の考え方

### ダイナミックプライシング
| 時期 | 価格倍率 | 理由 |
|------|---------|------|
| 平日（閑散期） | 0.7x | 稼働率を上げる |
| 週末 | 1.0x（基準） | 通常需要 |
| 連休・GW | 1.3-1.5x | 高需要 |
| お盆・年末年始 | 1.5-2.0x | 最高需要 |
| 地域イベント | 1.2-1.5x | 特需 |

### 最低宿泊日数
- 平日: 1 泊 OK
- 繁忙期: 2 泊以上
- 年末年始: 3 泊以上

## レビュー管理

### 目標
- 平均 4.8★ 以上（スーパーホスト維持）
- 返信率 100%（全レビューに返信）

### 低評価の原因 TOP5
1. 清掃の不備
2. Wi-Fi の速度
3. 説明文と実物の乖離
4. チェックイン手順の不明確さ
5. 近隣騒音

### 対策
- 清掃チェックリスト（`/operation-manual 清掃`）の徹底
- Wi-Fi 速度テスト（月 1 回）
- 写真の定期更新（季節ごと）
- チェックイン手順書の多言語化（`/guest-reply` で生成）

## 多言語対応

### 優先言語
1. 日本語（国内ゲスト）
2. 英語（欧米ゲスト）
3. 中国語 簡体字（中国本土ゲスト）
4. 韓国語（韓国ゲスト）
5. 中国語 繁体字（台湾・香港ゲスト）

### Claude Code での多言語生成
```
この文章を英語、中国語（簡体字）、韓国語に翻訳して:
「チェックインは15時からです。鍵はキーボックスにあります。」
```
