# De-Sloppify（品質クリーンアップ）

実装完了後に実行する「仕上げパス」スキルです。実装エージェントとは別の視点で、品質を一段引き上げます。

## コンセプト
「作る人」と「磨く人」を分ける。1人に両方やらせるより、2人の専門家（実装→クリーンアップ）の方が品質が高い。

## 実行手順

### Step 1: 変更範囲の特定
- `git diff` で今回の変更を全て確認
- 変更されたファイルをリストアップ
- 各ファイルの役割を理解

### Step 2: カテゴリ別チェック

#### HTML/CSS チェック（Web制作向け）
- [ ] 不要なインラインスタイルがないか
- [ ] クラス名が意味のある名前か（.div1 等の汎用名がないか）
- [ ] レスポンシブ対応: モバイル表示で崩れないか
- [ ] アクセシビリティ: alt属性、aria-label、コントラスト比
- [ ] 画像の最適化: 適切なサイズ、lazy loading
- [ ] メタタグ: title、description、OGP
- [ ] 不要なコメントアウトされたコードの除去
- [ ] console.log の除去
- [ ] フォントの読み込み順序と表示

#### JavaScript チェック
- [ ] 未使用の変数・関数
- [ ] エラーハンドリング漏れ
- [ ] パフォーマンス: 不要なDOM操作、重い処理
- [ ] イベントリスナーの重複登録

#### 全般チェック
- [ ] ファイルサイズが肥大化していないか
- [ ] コードの重複（同じ処理を2箇所以上で書いていないか）
- [ ] ハードコードされた値（色、URL、テキスト等）
- [ ] セキュリティ: XSS、外部スクリプトの信頼性

### Step 3: 自動修正
チェックリストで見つかった問題を修正する。修正は最小限に。構造を変えず、品質だけ上げる。

### Step 4: レポート
修正内容を簡潔に報告:
- 🔧 修正した項目数
- ⚠️ 手動確認が必要な項目
- ✅ 問題なかった項目

## 使い方
Web制作（/web-build）やサイト修正の後に `/de-sloppify` を実行。

## 注意
- 機能は追加しない。既存の品質を上げるだけ
- デザインの判断は変えない（ユーザーが決めたデザインは尊重）
- 大きな構造変更は提案のみ（実行しない）
