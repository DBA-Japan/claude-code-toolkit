# /guest-reply — ゲストメッセージ返信（多言語対応）

宿泊ゲストからの問い合わせ・予約確認・トラブル対応のメッセージを、適切なトーンと言語で返信ドラフトを生成します。

## 使い方

```
/guest-reply [メッセージ内容をペースト]
```

または

```
/guest-reply
（Claudeが内容を聞いてきます）
```

## 動作

### Step 1: メッセージ分析
- **言語判定**: 日本語 / 英語 / 中国語 / 韓国語 を自動判定
- **カテゴリ判定**: 予約確認 / チェックイン案内 / 設備質問 / トラブル / キャンセル / その他
- **緊急度判定**: 高（当日対応必要）/ 中（24h以内）/ 低（余裕あり）

### Step 2: 返信ドラフト生成
以下の原則に従って返信を生成:

1. **相手の言語で返信**（日本語のゲストには日本語、英語には英語）
2. **トーン**: 温かく丁寧。ホテルのフロントデスクではなく「家の主人が迎える」感覚
3. **簡潔**: 3-5文。長すぎない
4. **具体的**: 時間・場所・手順を明記
5. **ブランドの世界観**: 「暮らしに泊まる」の精神を反映

### Step 3: 出力
```
【言語】日本語
【カテゴリ】チェックイン案内
【緊急度】中

--- 返信ドラフト ---
（返信文）
--- ここまで ---

【補足】（必要に応じて注意点やフォローアップ提案）
```

## テンプレート（よくあるパターン）

### 予約確認
「ご予約ありがとうございます。[宿名]でお待ちしております。チェックインは[時間]から可能です。何かご不明な点がございましたらお気軽にお問い合わせください。」

### チェックイン案内
到着方法、鍵の受け取り、WiFi情報、緊急連絡先を含む

### 設備質問
具体的な設備情報（キッチン、洗濯機、駐車場、BBQ等）を回答

### トラブル対応
- まず謝罪と共感
- 具体的な解決策を提示
- フォローアップの約束

## 注意
- 料金交渉には応じない（「オーナーに確認します」と返す）
- キャンセルポリシーは各OTAの規約に従う
- 個人情報（電話番号等）はメッセージ内に含めない
