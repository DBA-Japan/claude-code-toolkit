# Context（コンテキスト管理ハブ）

コンテキストウィンドウの診断・コンパクト判断・トークン最適化を統合したハブスキル。

## 引数

$ARGUMENTS

## モード自動判定

ユーザーの発言に応じて自動でモードを選択する:

| ユーザーの意図 | モード | 旧コマンド |
|--------------|--------|----------|
| 「コンテキスト確認」「残量は？」「重い？」 | Budget（診断） | /context-budget |
| 「コンパクトすべき？」「いつcompactする？」 | Compact（判断） | /strategic-compact |
| 「軽くしたい」「MCP整理」「トークン減らしたい」 | Optimize（治療） | /token-optimize |
| 引数なし・意図が不明 | Budget → 問題あればOptimize提案 | — |

## Mode: Compact（判断）

「今 `/compact` すべきか？」を明確に答える。

### コンパクトの最適タイミング判定表
```
┌────────────────────────────────┬──────┬──────────────────────────┐
│ 今の作業フェーズ                │ 判定  │ 理由                      │
├────────────────────────────────┼──────┼──────────────────────────┤
│ リサーチ → 計画が固まった直後    │ ✅   │ 調査結果は圧縮しても残る     │
│ デバッグが解決した直後           │ ✅   │ 解決策はコードに反映済み      │
│ LP/サイトの1セクション完了後     │ ✅   │ 完成部分はファイルに保存済み   │
│ ディベート（議論）終了後         │ ✅   │ 結論をメモに残してから圧縮    │
│ 失敗したアプローチの後           │ ✅   │ 試行錯誤の履歴は不要         │
├────────────────────────────────┼──────┼──────────────────────────┤
│ 実装の途中（コードを書いている最中）│ ❌   │ 中間状態が消える             │
│ デバッグ中（原因を探っている最中） │ ❌   │ 排除情報が消え同じ場所を再調査 │
│ 複数ファイルを同時編集中         │ ❌   │ 全体像が失われる             │
│ 分析作業の途中                  │ ⚠️   │ 分析完了→結論メモ後なら OK   │
└────────────────────────────────┴──────┴──────────────────────────┘
```

### 何が残り、何が消えるか
- **残る**: CLAUDE.md, メモリファイル, Git状態, ディスク上のファイル, `/compact` に渡したメモ
- **消える**: 読んだファイル内容, 中間推論・試行錯誤, ツール呼び出し履歴, 会話の細かいニュアンス, 排除した選択肢の記録

### 手動 vs 自動コンパクト
- **手動**（`/compact メモ`）: タイミングを選べる＋メモを残せる → **品質が高い**
- **自動**（autocompact 閾値50%）: 保険として常にON → 作業途中で発動するリスクあり
- **ベストプラクティス**: 自動は保険、手動を区切りで積極的に使う

### 判定の出力パターン
- ✅ 今がいいタイミング → `/compact [引き継ぎメモ]` を具体的に提案
- ⏳ もう少し待つべき → 理由と「こうなったらcompactしよう」を提示
- 🤔 どちらでもよい → 両方のトレードオフを提示

## 重要な補足（全モード共通）

1. **体感が最強のサイン**: 数字よりも「回答がおかしくなった」という感覚が最も正確。迷わず `/compact` か新セッション
2. **トークン数は推定値**: 実際のシステムプロンプトの正確なサイズは外部から測れない。目安として使う
3. **最大の消費源は会話履歴**: 固定消費の最適化は「体質改善」。セッション中の消費は `/compact` で対処
4. **過度な最適化は逆効果**: CLAUDE.mdやメモリを削りすぎると判断品質が下がる。必要な情報は残す
5. **Opus 4.6 (1M) なら固定消費の影響は小さい**: 100万トークン中の数万は数%。Sonnet/Haikuの200Kでは影響大

## Claude Code Tips
| コマンド | 用途 | タイミング |
|---------|------|----------|
| `/cost` | セッションのトークン消費量・コスト表示 | 定期チェック |
| `/compact [メモ]` | 会話を要約圧縮。メモを残すと品質が劇的に向上 | 作業の区切り |
| `/clear` | セッション完全リセット | 別の作業に移る時 |
| `/model` | モデル切り替え（軽い作業→Sonnet、重い→Opus） | 作業内容に応じて |
