# /ota-listing — OTA 物件説明文の最適化

Airbnb / Booking.com / じゃらん 等の OTA（Online Travel Agency）に掲載する物件説明文を、検索上位・予約率向上を目指して最適化します。

## 使い方

```
/ota-listing [宿名] [プラットフォーム名]
```

例:
```
/ota-listing 東夢 Airbnb
/ota-listing ガーデンハウス道楽亭 Booking.com
```

## 動作

### Step 1: 情報収集
以下の情報を確認（なければ質問）:
- 宿の名前、所在地
- 定員、間取り、広さ
- 設備一覧（キッチン、WiFi、駐車場、BBQ等）
- 特徴・セールスポイント
- 周辺観光スポット
- 写真の説明（あれば）

### Step 2: プラットフォーム別最適化

#### Airbnb
- **タイトル**: 50文字以内。「築130年の古民家」「1日1組限定」等のキーワード
- **説明文構成**:
  1. キャッチコピー（1文）
  2. 宿の特徴（3-5項目）
  3. スペース詳細
  4. ゲストへのアクセス
  5. 周辺エリア
  6. その他の注意事項
- **言語**: 日本語 + 英語（両方生成）
- **SEOキーワード**: 地域名 + 宿泊スタイル + 特徴

#### Booking.com
- **物件名**: 簡潔かつ検索に強い
- **説明文**: 設備重視。箇条書き多め
- **ハウスルール**: 明確に記載

#### じゃらん / 楽天トラベル
- **プラン名**: 季節感 + 体験内容
- **説明文**: 日本語のみ。温かみのある表現

### Step 3: 出力
```
【プラットフォーム】Airbnb
【言語】日本語 / English

--- タイトル ---
（タイトル案 3つ）

--- 説明文 ---
（最適化済み説明文）

--- キーワード ---
（SEO用キーワード 10個）
```

## ベストプラクティス
- 写真のキャプションも最適化（「リビング」ではなく「築130年の梁が見える広々リビング」）
- 季節ごとに説明文を更新（春: 桜、夏: 海、秋: 紅葉、冬: 雪景色）
- レビューで褒められたポイントを説明文に反映
- 競合の上位リスティングを参考に差別化ポイントを強調
