# /review-reply — レビュー返信生成

宿泊ゲストのレビュー（星評価 + コメント）に対して、適切で誠実な返信を生成します。

## 使い方

```
/review-reply [レビュー内容をペースト]
```

## 動作

### Step 1: レビュー分析
- **評価**: 星の数（1-5）を判定
- **感情**: ポジティブ / ミックス / ネガティブ
- **言語**: 自動判定
- **キーポイント**: 褒めている点 / 改善要望 / クレーム を抽出

### Step 2: 返信生成ルール

#### 高評価（4-5星）
1. 感謝を述べる
2. 具体的に褒められた点に言及（「古民家の雰囲気を気に入っていただけて嬉しいです」）
3. 再訪を歓迎（季節の変化に触れる）
4. 3-4文で簡潔に

#### 中評価（3星）
1. 感謝を述べる
2. 良かった点に言及
3. 改善点について具体的な対応策を述べる（「ご指摘のWiFi速度について、回線の増強を検討しております」）
4. 次回の改善を約束

#### 低評価（1-2星）
1. 深くお詫びする
2. 具体的な問題点を認める（言い訳しない）
3. 既に実施した / これから実施する改善策を明記
4. 直接ご連絡いただける旨を伝える
5. 誠実さが最優先。テンプレ感を出さない

### Step 3: 出力
```
【元レビュー評価】★★★★☆
【感情】ポジティブ（一部改善要望あり）
【言語】英語

--- 返信ドラフト ---
（返信文）
--- ここまで ---

【内部メモ】（改善すべき点のサマリー。ゲストには見せない）
```

## 注意
- **返信の言語は相手に合わせる**
- **個人情報に言及しない**（「○○様のお子様が...」等は避ける）
- **他のゲストに見られることを意識**（公開レビューへの返信は営業ツール）
- **嘘をつかない**（「改善します」と書いたら本当に改善する）
