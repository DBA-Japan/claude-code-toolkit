# Audit（環境監査ハブ）

Claude Code環境の品質・セキュリティ・スキルを一括監査します。

## モード

| 引数 | 実行内容 |
|------|---------|
| なし（フルオーディット） | Harness → Security → Skills の順に全実行 → 統合レポート |
| `harness` | 環境スコアリングのみ（70点満点） |
| `security` | セキュリティスキャンのみ（A-F評価） |
| `skills` | スキル棚卸しのみ（Keep/Improve/Retire判定） |

引数: $ARGUMENTS

---

## Part 1: Harness Audit（環境スコアリング）

7カテゴリ × 10点 = 70点満点。各項目は0か2の二択（部分点なし）。

#### 1. CLAUDE.md 品質（/10）
- [ ] 存在する（+2）
- [ ] プロジェクト固有の情報がある（+2）
- [ ] ユーザーの役割・背景が記述されている（+2）
- [ ] 開発環境情報がある（+2）
- [ ] 500行以下（長すぎるとコンテキスト圧迫）（+2）

#### 2. フック構成（/10）
- [ ] SessionStart フックがある（+2）
- [ ] Stop フックがある（+2）
- [ ] PreToolUse セキュリティフックがある（+2）
- [ ] 全フックにタイムアウトが設定されている（+2）
- [ ] フックスクリプトが実行可能（chmod +x）（+2）

#### 3. メモリシステム（/10）
- [ ] MEMORY.md インデックスが存在する（+2）
- [ ] メモリファイルにfrontmatter（name/description/type）がある（+2）
- [ ] 不要なメモリが整理されている（重複なし）（+2）
- [ ] メモリの総サイズが適切（500KB以下）（+2）
- [ ] セッション間で情報が正しく引き継がれている（+2）

#### 4. スキル・コマンド（/10）
- [ ] ビジネスに直結するスキルがある（+2）
- [ ] スキルの説明が明確（+2）
- [ ] 重複スキルがない（+2）
- [ ] 使われていないスキルがない（+2）
- [ ] スキル間の連携が設計されている（+2）

#### 5. セキュリティ（/10）
- [ ] git のフックスキップをブロックするフックがある（+2）
- [ ] 機密ファイルへのアクセス監視がある（+2）
- [ ] 危険コマンドの検出がある（+2）
- [ ] MCP設定の安全性（信頼できるサーバーのみ）（+2）
- [ ] .env/credentials がgitignoreされている（+2）

#### 6. 自動化（/10）
- [ ] セッション保存が自動化されている（+2）
- [ ] コスト追跡が自動化されている（+2）
- [ ] 定期メンテナンスがスケジュールされている（+2）
- [ ] ヘルスチェックが自動実行されている（+2）
- [ ] バックアップが自動化されている（+2）

#### 7. コンテキスト管理（/10）
- [ ] コンテキストバジェット機能がある（+2）
- [ ] コンパクト戦略がある（手動or自動）（+2）
- [ ] MCP数が適切（10以下推奨）（+2）
- [ ] プラグイン数が適切（5以下推奨）（+2）
- [ ] 不要なMCPが無効化されている（+2）

**グレード基準**: S(63-70) / A(56-62) / A-(49-55) / B(42-48) / C(35-41) / D(28-34) / F(0-27)

**チェック方法**: `wc -l ~/CLAUDE.md`、`~/.claude/settings.json` を Read、`ls -la ~/.claude/hooks/`、`du -sh ~/.claude/projects/memory/`、`ls ~/.claude/commands/`、`ls ~/Library/LaunchAgents/`

---

## Part 2: Security Scan（セキュリティスキャン）

6領域をスキャンし、CRITICAL / WARNING / PASS / INFO で分類。

#### 1. CLAUDE.md — ハードコードされたシークレット検索
パターン: `sk-`, `api_key`, `token`, `secret`, `password`, `credential`, `allow all`, `unrestricted`

#### 2. settings.json — パーミッション設定
- [ ] `dangerouslySkipPermissions` の状態
- [ ] `allow` リストの範囲
- [ ] フックのタイムアウト設定（無制限がないか）

#### 3. フックスクリプト — コードインジェクション検査
危険パターン: `eval`, `curl`, `wget`, `exec`, `system(`, `chmod 777`
パーミッション: 755以下であること

#### 4. MCP設定 — 信頼性チェック
- [ ] HTTP（非HTTPS）のエンドポイント
- [ ] ローカルホスト以外のサーバー
- [ ] `skipSchemaValidation: true` の使用箇所

#### 5. メモリ — シークレット・インジェクション検査
シークレット: `sk-`, `api_key`, `token=`, `secret=`, `password=`
インジェクション: `ignore previous`, `disregard`, `new instructions`

#### 6. LaunchAgents — 自動化の安全性
- [ ] スクリプトのパーミッション
- [ ] 実行コマンドの安全性

**グレード基準**: A(CRITICAL 0, WARNING 0-2) / B(CRITICAL 0, WARNING 3-5) / C(CRITICAL 0, WARNING 6+) / D(CRITICAL 1-2) / F(CRITICAL 3+)

**注意**: シークレットの値そのものは出力に含めない。修正は必ずユーザー承認後。

---

## Part 3: Skill Stocktake（スキル棚卸し）

全スキルを5フェーズで監査。

#### Phase 1: インベントリ
`ls ~/.claude/commands/` の全ファイル → ファイルサイズ・最終更新日・カテゴリ（ビジネス/制作/管理/デザイン/その他）

#### Phase 2: 品質チェック（5基準）
| 基準 | チェック内容 |
|------|------------|
| 明確性 | 目的が1文で説明できるか |
| 完全性 | 手順が最初から最後まで書かれているか |
| 実用性 | 実際に使われているか |
| 鮮度 | 内容が最新か |
| 簡潔性 | 冗長な部分がないか |

#### Phase 3: 重複検出
似た目的のスキルがないか → 統合可能な組み合わせを提案

#### Phase 4: 判定
| 判定 | 意味 | アクション |
|------|------|----------|
| **Keep** | そのまま維持 | なし |
| **Improve** | 改善が必要 | 具体的な改善案を提示 |
| **Update** | 情報が古い | 更新箇所を指摘 |
| **Retire** | 不要 | 削除を提案（ユーザー承認後） |
| **Merge** | 統合候補 | 統合先を提案 |

#### Phase 5: サマリー
総スキル数・判定分布（Keep率）・カテゴリ分布・推奨アクションTOP3

---

## 実行プロセス

引数が `harness` / `security` / `skills` の場合 → 該当パートのみ実行してレポート出力。

引数なし（フルオーディット）の場合 → Part 1〜3を順番に実行し、最後に統合レポートを出力:

```
╔══════════════════════════════════════╗
║     Claude Code 総合監査レポート      ║
╠══════════════════════════════════════╣
║                                     ║
║  環境スコア:    XX/70 (グレード)     ║
║  セキュリティ:  X (グレード)         ║
║  スキル健全度:  XX% (Keep率)         ║
║                                     ║
║  総合評価: X                         ║
║                                     ║
║  改善推奨 TOP3:                     ║
║  1. [最も効果的な改善]              ║
║  2. [次に効果的な改善]              ║
║  3. [3番目の改善]                   ║
╚══════════════════════════════════════╝
```

**総合評価の算出**: 環境グレード(50%) + セキュリティグレード(30%) + Keep率(20%) の加重平均

## 注意事項
- スコアリングは実際のファイル・設定を確認して行う（推測しない）
- 改善アクションは具体的なコマンドやファイルパスを含める
- 推奨頻度: 月1回のフルオーディット
