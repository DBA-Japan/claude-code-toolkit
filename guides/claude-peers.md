# マルチインスタンス連携ガイド（claude-peers）

## claude-peers とは

claude-peers は、**同じマシン上で動いている複数のClaude Codeインスタンス間でリアルタイムに通信できる**MCPサーバーです。

通常、Claude Codeのインスタンスは互いに孤立しています。ターミナルウィンドウAで作業していることは、ターミナルウィンドウBのClaudeには見えません。claude-peers はこの「孤立」を解消します。

- 元リポジトリ: https://github.com/louislva/claude-peers-mcp

---

## ユースケース

### 1. 調査と実装の並列化

```
ウィンドウA（調査担当）          ウィンドウB（実装担当）
  /context で状況確認      →      調査結果を受け取って実装開始
  競合サイトを15社調査      →      調査中に別のタスクを進行
  結果をBに送信            →      結果が届いたら実装に組み込む
```

調査が終わるまで実装が止まっていた作業を、並列で進められます。

### 2. レビューと修正のループ

```
ウィンドウA（制作担当）          ウィンドウB（レビュー担当）
  Webページを実装           →      完成を知らせるメッセージを受信
  Bのレビュー結果を待つ           修正点をAに送信
  修正内容を受け取って修正  ←
```

### 3. バックグラウンドタスクの状態確認

長時間かかるタスク（大量ファイルの処理、APIの一括呼び出し等）を別ウィンドウで実行しながら、メインウィンドウで別の作業を続けられます。

### 4. 同じリポジトリへの協調作業（チームや複数セッション）

```
ウィンドウA: フロントエンドを担当
ウィンドウB: バックエンドを担当
→ お互いの状態を確認しながら、API仕様の変更を即座に共有
```

## セットアップ方法

### 前提条件

- **bun** が必要です（Node.js の代替ランタイム）

```bash
# bun がインストールされているか確認
bun --version

# インストールされていない場合
curl -fsSL https://bun.sh/install | bash
```

### インストール手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/louislva/claude-peers-mcp.git ~/claude-peers-mcp

# 2. 依存関係をインストール
cd ~/claude-peers-mcp && bun install
```

### MCP設定への登録

`~/.claude/settings.json` の `mcpServers` セクションに以下を追加します。

```json
{
  "mcpServers": {
    "claude-peers": {
      "command": "bun",
      "args": ["/Users/あなたのユーザー名/claude-peers-mcp/server.ts"]
    }
  }
}
```

このツールキットのインストーラー（`install.sh`）を使った場合は、「claude-peers をインストールする？」の質問で `y` を選ぶと自動設定されます。

### 設定の確認

Claude Codeを再起動してから、以下で動作確認します。

```
/mcp
```

`claude-peers` が表示されていれば設定完了です。

## 使い方

### set_summary — 自分の作業内容を宣言する

セッション開始時に、他のインスタンスに「自分が何をしているか」を伝えます。

```
set_summary を使って「Webサイトのヒーローセクションを実装中」と設定して
```

これにより他のインスタンスから `list_peers` したときに自分の状態が見えます。

---

### list_peers — 他のインスタンスを発見する

現在アクティブな他のClaude Codeインスタンス一覧を取得します。

```
list_peers で現在のインスタンス一覧を確認して
```

**出力例:**
```
ID: abc-123
Summary: 「Webサイトのヒーローセクションを実装中」
CWD: /Users/user/my-project
Last seen: 30秒前
```

scope（範囲）を指定できます:
- `machine` — 同じマシン上の全インスタンス（デフォルト）
- `directory` — 同じディレクトリのインスタンス
- `repo` — 同じGitリポジトリのインスタンス

---

### send_message — メッセージを送信する

```
ID が "abc-123" のインスタンスに
「ヒアリング分析が完了しました。主要課題は3点です: (1) ... (2) ... (3) ...」
とメッセージを送って
```

メッセージは相手のインスタンスがアクティブなセッションで受信します。

---

### check_messages — メッセージを確認する

他のインスタンスから届いたメッセージを確認します。

```
check_messages で届いているメッセージを確認して
```

メッセージが届いていると、Claude Code が自動的に通知し、内容を表示します。

---

## 実践的なワークフロー例

### パターン1: 大規模調査 + 即時実装

```bash
# ターミナルA（調査）
claude
```

```
# ウィンドウA での指示
set_summary を使って「競合サービス20社の料金調査を実行中」と設定
（調査が完了したら）
list_peers で実装担当のインスタンスを探して、結果を send_message で送信して
```

```bash
# ターミナルB（実装）
claude
```

```
# ウィンドウB での指示
set_summary を使って「料金ページのUIを実装待機中」と設定
check_messages で調査結果の到着を待って、届いたら即座に料金テーブルを実装して
```

---

### パターン2: 長時間タスクの監視

```bash
# ターミナルA（バックグラウンド処理）
claude
```

```
# 1000件のCSVデータを処理して、完了したら list_peers でメインインスタンスを探して
# 「処理完了: 成功945件、失敗55件」と send_message で通知して
```

```bash
# ターミナルB（メイン作業）
claude
```

メインウィンドウで別の作業をしながら、バックグラウンド処理の完了通知を待てます。

---

## 注意事項

- **同じマシン上でのみ動作します。** リモートマシン間の通信はできません。
- **セッション間の共有メモリではありません。** メッセージはリアルタイム送受信です。送信した時点でもう一方がアクティブでないと届きません。
- **bun のアップデートに注意。** bun をアップデートすると設定が変わる場合があります。動かなくなった場合は `~/claude-peers-mcp` で `bun install` を再実行してください。
