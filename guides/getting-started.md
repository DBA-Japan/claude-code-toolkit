# はじめてのClaude Code Toolkit

## Claude Code とは

Claude Code は、ターミナル上で動くAnthropicの公式CLIエージェントです。「チャットで質問する」のではなく、「一緒に作業するパートナー」として機能します。ファイルを読み書きし、コマンドを実行し、コードベース全体を横断して理解しながら複雑なタスクをこなせる点が最大の特徴です。プログラミングの経験がなくても、自然な日本語で指示するだけで本格的なWebサイト制作、データ分析、業務自動化が実現できます。

---

## このツールキットで何が変わるか

インストール直後のClaude Codeは「優秀だが何も知らない新人」です。あなたのことを知らず、毎回ゼロから始まります。このツールキットは以下の3点を解決します。

| 課題 | ツールキットの解決策 |
|------|---------------------|
| セッションをまたぐと文脈が消える | メモリシステム + セッション自動保存 |
| Claudeに自分のことを毎回説明する | CLAUDE.md（常時読み込まれる個人設定） |
| 繰り返すタスクを毎回手動でやる | スキル（/コマンド）で一発実行 |
| 危険な操作に気づかない | Hookによる自動ガード |

## インストール後の最初の5ステップ

### Step 1: `/context` で環境確認

Claude Codeを起動したら、まず現在のコンテキスト状態を確認します。

```
/context
```

「Budget（診断）」モードが自動で起動し、現在のメモリ使用量、固定消費している項目、健康度が表示されます。インストール直後は問題ないはずですが、今後の基準値として把握しておきましょう。

**なぜこれが重要か:** コンテキストウィンドウは有限です。どこに何が消費されているかを把握しておくことで、後から「回答が雑になってきた」と感じたときに原因を特定できます。

---

### Step 2: `/audit` で品質チェック

```
/audit
```

インストールが正しく完了しているかを3つの観点でスキャンします。

- **Harness（70点満点スコアリング）:** Hooks・スキル・CLAUDE.md・メモリが正しくセットアップされているか
- **Security（脆弱性スキャン）:** 秘密鍵・.envファイルの露出リスク、危険な設定がないか
- **Skills（棚卸し）:** どのスキルが使えるか一覧で確認

スコアがA未満の場合は、表示される改善提案を順番に実行してください。

---

### Step 3: CLAUDE.md を自分用にカスタマイズ

`~/CLAUDE.md` はClaude Codeが**毎回セッション開始時に必ず読む**設定ファイルです。ここに書いたことはすべてのセッションで有効になります。

インストーラーが生成したテンプレートを開いて、以下の項目を自分用に編集してください。

```bash
open ~/CLAUDE.md
```

**カスタマイズのポイント:**

- **ユーザーについて:** 名前・役割・使用目的
- **コマンドの自動提案ルール:** よく使うスキルをどんな文脈で提案してほしいか
- **絶対ルール:** Claudeに「これだけは自動でやるな」と伝えること（例: 本番デプロイは確認してから）
- **意思決定パターン:** 「直して」と言ったらすぐ実行・確認不要、など

**Tips:** CLAUDE.mdは長くなりすぎるとコンテキストを大量消費します。1,000行を超えてきたら `/context optimize` で圧縮を検討してください。

---

### Step 4: メモリシステムを理解する（`/learn-eval`）

メモリは `~/.claude/projects/{プロジェクトパス}/memory/` に保存されるMarkdownファイル群です。構造は以下の通りです。

```
memory/
├── MEMORY.md          ← インデックス（200行以内に保つ）
├── topic-A.md         ← 詳細ファイル（MEMORY.mdからリンク）
├── topic-B.md
└── ...
```

セッション中に「これは覚えておきたい」と思ったことがあれば:

```
/learn-eval
```

と打つだけです。セッション中に学んだTipsや解決パターンを振り返り、品質評価（Save/Improve/Absorb/Drop）をしてからメモリに保存します。重複チェックと既存ファイルへの追記判定も自動で行われます。

**なぜMEMORY.mdを200行以内に保つか:** MEMORY.mdは毎回読み込まれます。肥大化するとコンテキストをどんどん消費します。詳細は別ファイルに切り出し、MEMORY.mdはインデックス（目次）として機能させるのが原則です。

---

### Step 5: 最初のスキルを試す（`/aside` が一番簡単）

スキルの中で最も使いやすいのが `/aside` です。

```
/aside
```

現在の作業のコンテキストを保持したまま、サイドタスクを処理して自動的に元の作業に戻ります。例えば「Webサイトを作っている途中で、ちょっとこのメールの返信文を書いてほしい」という場面で使います。

スキルは `~/.claude/commands/` にある `.md` ファイルです。どんなスキルが使えるか確認するには:

```
ls ~/.claude/commands/
```

## 必ず覚えておくべきTips

### `/compact` の使い方

会話が長くなると、Claudeの回答品質が少しずつ下がります（コンテキストウィンドウが埋まるため）。品質低下のサインは:

- 回答が短くなった
- 指示を忘れている
- 同じことを2回聞いてくる

これを感じたら:

```
/compact
```

会話の要点を圧縮して「新鮮な状態」に戻します。重要な作業中は `/compact` の前に「ここまでの進捗: ○○」と一言メモを付けると、圧縮後も文脈が維持されます。

---

### モデル選択の基準

Claude Codeは複数のモデルを使い分けられます。タスクに応じて適切なモデルを指定することでコストと品質のバランスが取れます。

| タスク | 推奨モデル | 理由 |
|--------|----------|------|
| ファイル検索・簡単な質問 | Haiku | 軽い作業は安いモデルで |
| 実装・制作・通常の対話 | Sonnet | ベストバランス |
| 複雑な設計・多角的な判断 | Opus | 深い思考が必要な場面 |

迷ったらSonnetで始めて、不十分ならOpusにエスカレートするのが基本戦略です。

---

### エージェントの使い方

Claude Codeは「サブエージェント」を使って並列作業ができます。自分で作業しながら別のエージェントに調査・実装・レビューを依頼できます。

エージェントを使うときは、以下の5項目を必ず伝えてください（伝えないと質が落ちます）:

1. **目的:** このエージェントに何を達成してほしいか（1文）
2. **背景:** なぜその調査/作業が必要か
3. **参照:** 関連ファイルパス、URL
4. **期待出力:** テーブル/リスト/レポートなど形式を指定
5. **成功基準:** 何が返ってきたら「十分」か

結果が不十分な場合は追加コンテキストを渡して再実行（最大3回）。

## よくある質問

**Q: CLAUDE.mdとMEMORY.mdの違いは？**
A: CLAUDE.mdは「Claudeへの指示書」です。どう動いてほしいかのルールを書きます。MEMORY.mdは「知識の蓄積」です。プロジェクトの文脈・過去の決定・Tips などを保存します。

**Q: スキルを自分で作れる？**
A: はい。`~/.claude/commands/` に `.md` ファイルを置くだけです。詳しくは `guides/skill-design.md` を参照してください。

**Q: Hookって何？**
A: セッション開始・終了・コマンド実行などのイベントで自動実行されるシェルスクリプトです。詳しくは `guides/hooks-explained.md` を参照してください。
