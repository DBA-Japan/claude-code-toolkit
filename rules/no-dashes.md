# ダッシュ・ハイフン全廃ルール

表示テキストおよびCSS装飾線でダッシュ系記号を使用禁止。AI生成テキストの典型サインで即バレするため、強い忌避を表明した複数の利用者により絶対ルールとして固定。

## 禁止対象

### 1. テキスト記号

- `—` (em dash, U+2014)
- `–` (en dash, U+2013)
- `ーー` (連続長音)
- `――` (連続全角ダッシュ)
- `--` (連続ハイフン)
- ハイフン1個でも文中区切り（`Word - Word`）

### 2. CSS装飾線（テキスト記号と同等NG）

- `::before/::after` で描く短線スタイル（`width:Npx;height:1px;background:...`）
- ラベル前後・章番号前後・カテゴリ前後の装飾線
- 例: `.sequence__label::before` `.sequence__num::before` 等

特に**箇条書き代わり**として使う形（`— ポイント1` / `ーーそう言われる。`）が一番危険。

## 唯一の例外

**物語的eyebrow**（CTA直前等の儀式的タイポ）の両端の装飾線のみOK。

- 例: `dba-dna.html` の `.cta-eyebrow::before/::after`（"DON'T BE AFRAID" 両端の線）
- 「ダッシュの代用」ではなく「儀式的な額装」として機能する唯一の場面
- **他の場所でこの形を流用しない**

## その他の例外（技術仕様）

- HTMLコメント・CSSコメント
- CSS変数名 (`--sumi-soft` 等)
- 矢印アイコン (`→` `←` `↗` 等の Unicode 矢印)
- `<hr>` 全幅区切り（footer rule 等の通常区切り）

## 代替表現

| 用途 | 代替 |
|---|---|
| 区切り | `、` `。` `｜` `／` |
| 強調 | 改行 / 太字 / 1文に分ける |
| リスト | `<ul><li>` または番号 / 改行 |
| 補足 | `（カッコ）` |

## 自己チェック手順（生成後に必ず実行）

```bash
# 1. テキスト記号検出
grep -nE "(—|–|ーー|――|--)" target.html

# 2. CSS装飾線検出
grep -nE "::before|::after" target.html
# ヒット箇所をレビュー → 短線スタイル(width:Npx;height:1px;background:...)が
# ラベル/番号前後に使われていたら全部削除
```

矢印・技術仕様・物語的eyebrow例外以外でヒットしたら全て修正する。

## 適用範囲

- Webサイト本文・コピー・タイトル・OGP メタ
- SNS投稿（LinkedIn / X 等）
- 営業メール
- スライド・PPTX 表示テキスト
- 全ての**ユーザー目に触れる表示テキスト**
